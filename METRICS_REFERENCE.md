# Metrics Reference Guide

Quick reference for all metrics generated by the pipeline and how they sync to the paper.

## Metric Generation Flow

```
Phase 1: PlantSwarm
    ↓ generates
    plantswarm_metrics.json + traces/
    ↓
Phase 2: Baselines, Ablations, Calibration, Routing
    ↓ generates
    baseline_results.json, calibration_report.json, routing_analysis.json
    ↓
Phase 3: OBSERVE Training
    ↓ generates
    observe/checkpoints/observe_final.pt + training_history.json
    ↓
Phase 4: OBSERVE Evaluation (OOD)
    ↓ generates
    observe_evaluation.json
    ↓
Phase 5: Sync to LaTeX
    collect_metrics.py aggregates all JSONs
    sync_latex_metrics.py fills in paper placeholders
    ↓
plantswarm/latex/auto_*.tex (6 auto-generated table fragments)
    ↓
latexmk generates acl_latex.pdf with all metrics filled in
```

Main results across all tasks and benchmarks:

```json
{
  "T1": {
    "macro_f1": 0.892,
    "ece": 0.085,
    "tpcp": 645.3,
    "agent_accuracy": 0.89,
    "early_termination_rate": 0.12,
    "backtrack_rate": 0.18,
    "mean_path_length": 3.4
  },
  "T2": { ... },
  "T3": { ... },
  "T4": { ... },
  "T5": { ... },
  "by_benchmark": {
    "plantvillage": { "T1": {...}, "T2": {...}, ... },
    "plantdoc": { ... },
    "plantwild": { ... }
  },
  "metadata": {
    "subset": 10000,
    "date": "2026-05-01",
    "backend": "vllm",
    "model": "Qwen/Qwen2.5-VL-3B-Instruct"
```

**Key fields for paper:**
- `plantswarm_metrics.T3.macro_f1` → Table 1, PlantSwarm T3 column
- `plantswarm_metrics.T3.ece` → Table 1, PlantSwarm ECE
- `plantswarm_metrics.T3.tpcp` → Table 1, PlantSwarm TPCP
- `by_benchmark.plantvillage.T3.macro_f1` → Benchmark-specific results

## LaTeX Auto-Generation

### Input Files (from experiments)
```
results/plant_village_tfds/
├── plantswarm_metrics.json ─┐
├── baseline_results.json    ├─→ collect_metrics.py ──→ unified_metrics.json
├── calibration_report.json  │
├── routing_analysis.json    │
└── ablation_metrics_*.json ─┘
```

### Processing
```
unified_metrics.json + sync_latex_metrics.py ──→ plantswarm/latex/auto/*.tex
```

### Output LaTeX Files

1. **auto_metrics.tex** (inline macros)
   ```latex
   \providecommand{\AutoMacroFOneTThree}{0.892}
   \providecommand{\AutoECETThree}{0.0823}
   \providecommand{\AutoTPCPTThree}{645.3}
   ```
   **Used in paper text:** `\AutoMacroFOneTThree` renders as "0.892"

2. **auto_table_main_results.tex** (Table 1)
   ```latex
   \begin{table*}...\end{table*}  % Full table with filled metrics
   ```
   **Included in acl_latex.tex:** `\input{auto/auto_table_main_results.tex}`

3. **auto_table_ablation_results.tex** (Table 3)
4. **auto_table_predictions.tex** (Table 4)
5. **auto_table_mechanisms.tex** (Table 5)
6. **auto_table_budget.tex** (Table 6, optional)

## Metric Dependencies

```
PlantSwarm (Phase 1)
  ├─ plantswarm_metrics.json ──→ Table 1 PlantSwarm row
  └─ traces/ ──→ OBSERVE training data

Baselines (Phase 2a)
  └─ baseline_results.json ──→ Table 1 baseline rows

Ablations (Phase 2b)
  └─ ablation_metrics_*.json ──→ Table 3 rows

Calibration (Phase 2c)
  └─ calibration_report.json ──→ Calibration section, ECE metrics

Routing (Phase 2d)
  └─ routing_analysis.json ──→ Table 4 (P1-P4)

OBSERVE Training (Phase 3)
  └─ training_history.json ──→ Figure: training curves

OBSERVE Evaluation (Phase 4)
  └─ observe_evaluation.json ──→ Table 2 OBSERVE rows

All above ──→ sync_latex_metrics.py ──→ plantswarm/latex/auto_*.tex
```

## Common Metric Issues

### Issue: All baselines show "---" in paper
**Cause:** `baseline_results.json` not found  
**Fix:**
```bash
python scripts/run_baselines.py --config configs/plant_village_tfds.yaml
python scripts/sync_latex_metrics.py --results-dir results/plant_village_tfds --latex-dir plantswarm/latex
```

### Issue: Ablation rows are blank
**Cause:** `ablation_metrics_*.json` files missing  
**Fix:**
```bash
python scripts/run_ablations.py --config configs/plant_village_tfds.yaml
```

### Issue: OBSERVE ECE in paper is still "---"
**Cause:** Phase 3/4 evaluation not run yet  
**Fix:**
```bash
python scripts/train_observe.py --traces results/plant_village_tfds/traces/plantswarm_traces.jsonl
# Then evaluate if running on OOD data
python scripts/evaluate_observe.py --model observe/checkpoints/observe_final.pt --traces results/plantwild/traces/plantswarm_traces.jsonl
```

### Issue: LaTeX won't compile
**Cause:** auto_*.tex files have syntax error  
**Debug:**
```bash
pdflatex plantswarm/latex/acl_latex.tex | head -50  # Shows first error
# Or: latexmk -pdf -interaction=nonstopmode plantswarm/latex/acl_latex.tex
```

## Extending the Metrics System

To add new metrics:

1. **Add to experiment output JSON**
   ```python
   # In your script
   results = {..., "new_metric": 0.789}
   with open("results/my_output.json", "w") as f:
       json.dump(results, f)
   ```

2. **Update `collect_metrics.py`**
   ```python
   def collect_my_metrics(results_dir: Path) -> dict:
       data = load_json_safe(results_dir / "my_output.json", {})
       return {"my_metrics": data}
   ```

3. **Update `sync_latex_metrics.py`**
   ```python
   # In LaTeXGenerator.generate_metrics_macros():
   add_macro("MyMetric", str(self.metrics.get("my_metrics", {}).get("value")))
   ```

4. **Use in paper**
   ```latex
   My metric: \providecommand{\MyMetric}{0.789}
   ```
